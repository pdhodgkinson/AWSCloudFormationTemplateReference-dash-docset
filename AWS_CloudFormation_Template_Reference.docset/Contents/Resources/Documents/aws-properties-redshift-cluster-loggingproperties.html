Amazon Redshift LoggingProperties

Use the LoggingProperties property of the AWS::Redshift::Cluster resource to configure audit log files, containing information such as queries and connection attempts, for the cluster.

Syntax

JSON

Copy
{
  "BucketName" : String,
  "S3KeyPrefix" : String
}

YAML

Copy
BucketName: String
S3KeyPrefix: String

Properties

For more information and property constraints, see EnableLogging in the Amazon Redshift API Reference.

BucketName

The name of an existing S3 bucket where the log files are to be stored.

Required: Yes

Type: String

Update requires: No interruption

S3KeyPrefix

The prefix applied to the log file names.

Required: No

Type: String

Update requires: No interruption
