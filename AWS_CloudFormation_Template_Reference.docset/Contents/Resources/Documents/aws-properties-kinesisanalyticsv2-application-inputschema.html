Amazon Kinesis Data Analytics Application InputSchema

The InputSchema property type for a SQL-based Kinesis Data Analytics application to describe the format of the data in the streaming source, and how each data element maps to corresponding columns created in the in-application stream.

InputSchema is a property of the Input property.

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "RecordEncoding" : String,
  "RecordColumns" : [ RecordColumn, ... ],
  "RecordFormat" : RecordFormat
}

YAML


RecordEncoding: String
RecordColumns: 
  - RecordColumn
RecordFormat: RecordFormat

Properties

RecordEncoding

Specifies the encoding of the records in the streaming source. For example, UTF-8.

Required: No

Type: String

Update requires: No interruption

RecordColumns

A list of RecordColumn objects.

Required: Yes

Type: List of RecordColumn

Update requires: No interruption

RecordFormat

Specifies the format of the records on the streaming source.

Required: Yes

Type: RecordFormat

Update requires: No interruption
