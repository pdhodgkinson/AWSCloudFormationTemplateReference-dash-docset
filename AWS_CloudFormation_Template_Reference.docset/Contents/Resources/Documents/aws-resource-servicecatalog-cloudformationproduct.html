AWS::ServiceCatalog::CloudFormationProduct

Creates the specified product for AWS Service Catalog. For more information, see CreateProduct in the AWS Service Catalog Developer Guide.

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "Type" : "AWS::ServiceCatalog::CloudFormationProduct",
  "Properties" : {
    "Owner" : String,
    "SupportDescription" : String,
    "Description" : String,
    "Distributor" : String,
    "SupportEmail" : String,
    "AcceptLanguage" : String,
    "SupportUrl" : String,
    "Tags" : [ Resource Tag, ... ],
    "Name" : String,
    "ProvisioningArtifactParameters" : [ ProvisioningArtifactProperties, ... ]
  }
}

YAML

Type: "AWS::ServiceCatalog::CloudFormationProduct"
Properties:
  Owner: String
  SupportDescription: String
  Description: String
  Distributor: String
  SupportEmail: String
  AcceptLanguage: String
  SupportUrl: String
  Tags: 
    - Resource Tag
  Name: String
  ProvisioningArtifactParameters: 
    - ProvisioningArtifactProperties

Return Values

Ref

When you pass the logical ID of an AWS::ServiceCatalog::CloudFormationProduct resource to the intrinsic Ref function, the function returns the ID of the provisioning artifact, such as prod-nd24wbqkm4pju.

For more information about using the Ref function, see Ref.

Fn::GetAtt

Fn::GetAtt returns a value for a specified attribute of this type. The following are the available attributes.

ProductName

The name of the product.

ProvisioningArtifactIds

The IDs of the provisioning artifacts.

ProvisioningArtifactNames

The names of the provisioning artifacts.

For more information about using Fn::GetAtt, see Fn::GetAtt.
